## DEPLOYMENT ARCHITECTURE

### Seeding Process

Each department follows this **agent specification pattern**:

```markdown
---
name: [Agent Title]
description: [Expert description + key capabilities]
color: [Department color, hex]
emoji: [Relevant emoji]
vibe: [Cultural tagline]
---

# [Agent Header with emoji]

## 🧠 Your Identity & Memory
[Character background, 10+ years experience, superpower, core beliefs]

## 🎯 Your Core Mission
[Primary mission statement and deliverables]

## 🚨 Critical Rules You Must Follow
[5-8 non-negotiable rules for this role]

## 📋 Your Technical Deliverables
[Detailed capability breakdown with:
  - Specific processes
  - Key responsibilities
  - Deliverable types
  - Tools & technologies
  - Templates & examples]
```

### Agent Characteristics

Each agent includes:
- ✅ **Detailed Identity**: 10+ years experience, proven track record
- ✅ **Core Mission**: Clear purpose aligned to department strategy
- ✅ **Critical Rules**: 5-8 non-negotiable rules preventing common failures
- ✅ **Technical Deliverables**: Specific processes, responsibilities, tools
- ✅ **Templates & Examples**: 2-3 real-world templates per agent

### Deployment Features

- **Platform Catalog**: All agents available to all tenants at signup
- **Selective Onboarding**: Tenants choose which departments/agents to deploy
- **Customization**: Agents adapt to tenant industry, size, stage
- **Scalability**: One agent → Many agents (manager + ICs for scaling)
- **Audit Logging**: All agent deployments tracked and auditable

## KEY IMPLEMENTATION NOTES

### Database Schema
All agents are seeded as platform-level catalog entries:
- `PoolAgent` records with `tenantId = null` (platform-level)
- `PoolDepartment` records for all 25 departments
- `IndustryPackage` entries grouping agents by industry + tier
- `Agent` records created on tenant deploy for selected agents

### Tenant Onboarding Flow (Updated)
1. **Company** step: Capture industry (new enum: 10 values including HEALTHCARE, FINANCE, etc.)
2. **Plan** step: Select tier (FREE, PRO, ENTERPRISE)
3. **Industries/Packages** step (NEW): Recommend agents by industry + tier
4. **Template** step: Legacy fallback for tenants without packages
5. **Review**: Confirm selected departments/agents before deploy
6. **Deploy**: Upsert all selected agents to tenant's account

### Feature Flags
- `ADMIN_POOL_V2=true`: Enable new pool management UI
- `ONBOARDING_USE_INDUSTRY_PACKAGES=true`: Enable package step in wizard
- Per-tenant overrides available for gradual rollout
