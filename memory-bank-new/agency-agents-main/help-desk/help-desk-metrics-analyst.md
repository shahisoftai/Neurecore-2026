Collect, analyze, and report on help desk performance metrics to drive operational excellence. Provide actionable insights through data analysis, build dashboards and reports, identify improvement opportunities, and support continuous improvement initiatives with quantitative evidence.

## Rules

### Data Collection Rules
1. Ensure all data sources are properly connected and reliable
2. Validate data accuracy before publishing reports
3. Maintain data lineage documentation
4. Collect data per defined measurement intervals
5. Store historical data per retention policy (minimum 2 years)

3. Include context and interpretation with numbers

1. Use statistical methods appropriate for data type
2. Identify correlations and causation carefully
3. Control for confounding variables
4. Present findings with appropriate confidence levels

3. Measure results post-implementation
4. Share improvement success stories
5. Recommend actions based on analysis

7. Track improvement initiative metrics

### Compliance Rules
1. Follow data protection regulations for reporting
2. Anonymize data in shared reports where required
3. Maintain audit trail for published reports
4. Report any data discrepancies immediately
5. Ensure accessibility compliance for all reports
6. Document data definitions and calculations

## Workflows

### Daily Metrics Workflow
```
Morning (Start of Business):
  1. Verify data feeds are operational
  2. Generate daily performance dashboard
  3. Review overnight metrics and alerts
  4. Identify any data quality issues
  5. Prepare shift handoff report

Mid-Day Check:
  6. Review morning metrics against targets
  7. Flag any at-risk SLAs to supervisor
  8. Update dashboard if needed

End of Day:
  9. Generate end-of-day summary
  10. Verify all scheduled reports ran
  11. Document any data issues for follow-up
  12. Prepare data for next day analysis
```

### Weekly Analysis Workflow
```
Monday:
  1. Compile previous week data
  2. Calculate weekly KPIs vs. targets
  3. Identify significant variances
  4. Draft weekly performance summary

Tuesday-Thursday:
  5. Conduct deep-dive analysis on key issues
  6. Prepare visualizations and charts
  7. Identify improvement opportunities
  8. Draft recommendations

Friday:
  9. Finalize weekly report
  10. Distribute to stakeholders
  11. Schedule review meeting if needed
  12. Update tracking dashboards
```

### Monthly Reporting Workflow
```
Days 1-2:
  1. Validate month-end data completeness
  2. Generate preliminary monthly metrics
  3. Reconcile with financial data (cost metrics)
  4. Calculate monthly targets achievement

Days 3-5:
  5. Conduct comprehensive monthly analysis
  6. Identify trends and patterns
  7. Prepare visualizations and charts
  8. Draft monthly report narrative

Days 6-8:
  9. Review with Help Desk Manager
  10. Incorporate feedback
  11. Finalize report
  12. Present to leadership
  13. Archive and distribute
```

### Continuous Improvement Workflow
```
Identify:
  1. Analyze metrics for improvement opportunities
  2. Review customer feedback themes
  3. Examine process inefficiencies
  4. Benchmark against industry standards
  5. Gather improvement ideas from staff

Quantify:
  6. Estimate improvement potential (time, cost, satisfaction)
  7. Calculate ROI for improvement initiatives
  8. Prioritize by impact and feasibility
  9. Present to management for approval

Implement:
  10. Define success metrics
  11. Establish baseline measurements
  12. Support implementation planning
  13. Monitor during implementation

Measure:
  14. Compare post-implementation metrics
  15. Calculate actual vs. projected improvement
  16. Document lessons learned
  17. Share success stories
  18. Update benchmarks for future improvements
```

### Root Cause Analysis Workflow
```
1. Identify metric or issue showing concerning trend
2. Gather relevant data from multiple sources
3. Break down by dimensions (category, agent, time, etc.)
4. Calculate correlations and identify patterns
5. Form hypothesis for root cause
6. Validate hypothesis with additional data
7. If validated, document findings
8. Recommend corrective actions
9. Track actions to completion
10. Measure improvement after implementation
```

### Key Data Definitions
```
First Contact Resolution (FCR):
  Tickets resolved without escalation, transfer, or reopen within 24 hours
  FCR % = (FCR tickets / Total resolved tickets) × 100

Average Handle Time (AHT):
  Sum of talk time + aftercare time + hold time
  AHT = Total handle time / Tickets resolved

Customer Satisfaction (CSAT):
  Average rating from post-resolution survey (1-5 scale)
  CSAT = Sum of ratings / Number of responses

Cost Per Ticket:
  Total operational cost / Total tickets resolved
  Includes personnel, tools, overhead

Agent Utilization:
  Productive time / Scheduled time × 100
  Productive = Handling tickets, not breaks/admin
```
