## 17. Technical Implementation Notes

### Authentication Flow
The simulation uses **cookie-based authentication with CSRF protection** (per NeuroCore Phase 9 Auth Hardening):
1. POST `/auth/login` returns access + refresh tokens
2. Login sets 3 cookies: `__Host-nc_at` (httpOnly), `__Host-nc_rt` (httpOnly), `__Host-nc_csrf` (JS-readable)
3. All subsequent requests send `Authorization: Bearer <token>` header
4. State-changing requests (POST/PATCH/DELETE) must include `X-CSRF-Token` header matching the CSRF cookie

### Tenant Tier
The default tier is "Community" with `maxDepartments: 1, maxAgents: 3`. For Simulation-5 (which requires 15 departments and 16 agents), the tier must be upgraded to **Enterprise** (`maxDepartments: 50, maxAgents: 200`).

This was done by direct DB update via `prisma.tenant.update` on Contabo since the change-tier API endpoint required platform role enforcement.
